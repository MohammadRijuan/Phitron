Discuss The Scope of var, let, and const

  • Answer: Variables declared with var are function-scoped. This means they are only accessible within the function they are declared in. If declared outside any function, they become global variables.
  • Answer: let is a modern way to declare variables in JavaScript, introduced in ECMAScript 6 (ES6). It is preferred over var for several reasons due to its block-scoping and lack of hoisting issues.
  • Answer: const is another modern way to declare variables in JavaScript, also introduced in ECMAScript 6 (ES6). It is used to declare variables that are meant to be constants, meaning their value should not be reassigned after their initial assignment.

Tell us the use cases of null and undefined

  • Answer: null is a special value in JavaScript that represents the intentional absence of any object value. It is used to explicitly indicate that a variable should have no value.
  • Answer: undefined is a primitive value in JavaScript that represents the absence of a value or the uninitialized state of a variable. It is used by the JavaScript engine to indicate that a variable has been declared but not yet assigned a value.

What do you mean by REST API?

Answer: A REST API (Representational State Transfer Application Programming Interface) is a set of rules and conventions for building and interacting with web services. REST APIs use HTTP requests to perform CRUD (Create, Read, Update, Delete) operations on resources.
